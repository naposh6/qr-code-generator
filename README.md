# GenerQR — Генератор динамічних QR-кодів

[cite_start]Цей проєкт реалізовано як фінальну роботу, основною метою якої є демонстрація навичок написання чистого коду, застосування патернів проєктування та дотримання принципів ООП[cite: 1, 2, 3].

## 📋 Опис функціоналу
[cite_start]Програма дозволяє створювати QR-коди для різних типів даних[cite: 14]:
* **Посилання та текст**: Пряме кодування введених даних.
* [cite_start]**Медіафайли (фото/відео)**: Автоматичне завантаження файлу на сервер та генерація QR-коду з посиланням на цей локальний ресурс[cite: 20].
* **Система редиректів**: Використання коротких посилань для перенаправлення користувача (механізм для подальшого збору аналітики).
* [cite_start]**Історія та керування**: Перегляд останніх генерацій у зручній таблиці з можливістю повного видалення записів з бази даних[cite: 20].

## 🚀 Процес запуску локально
1. **Клонування репозиторію**: 
   [cite_start]`git clone https://github.com/naposh6/qr-code-generator.git` [cite: 28]
2. **Встановлення залежностей**: 
   Виконайте `composer install` для завантаження бібліотеки `endroid/qr-code`.
3. **Налаштування БД**: 
   Створіть MySQL базу даних та імпортуйте таблиці (структуру можна знайти в `src/Repositories/QrRepository.php`).
4. **Конфігурація**: 
   Вкажіть ваші дані підключення до БД у файлі `src/Core/Database.php`.
5. **Веб-сервер**: 
   Проєкт протестовано на WAMP/XAMPP. Переконайтеся, що модуль `mod_rewrite` увімкнено для коректної роботи `.htaccess`.

## 💎 Programming Principles (Принципи програмування)
[cite_start]У проєкті реалізовано наступні принципи[cite: 32, 33]:
* **Single Responsibility (SRP)**: Кожен клас виконує лише одну функцію (наприклад, `FileService` відповідає тільки за роботу з файловою системою).
* **Open/Closed Principle**: Система відкрита для розширення (можна додавати нові моделі контенту) і закрита для модифікації базового ядра.
* **DRY (Don’t Repeat Yourself)**: Логіка роботи з базою даних централізована в одному класі.
* **KISS (Keep It Simple, Stupid)**: Архітектура проєкту побудована максимально просто для полегшення підтримки без використання важких фреймворків.
* **Separation of Concerns**: Чіткий поділ на рівень доступу до даних (Repository), бізнес-логіку (Services) та відображення (Views).

## 🏗️ Design Patterns (Патерни проєктування)
[cite_start]В коді застосовано 3 ключові патерни[cite: 34, 35]:
1. **Singleton** (`src/Core/Database.php`): Забезпечує єдине спільне підключення до бази даних для всього додатка.
2. **Factory Method** (`src/Factories/QrContentFactory.php`): Відповідає за створення правильного об'єкта контенту (URL, Text або File) на основі вводу користувача.
3. **Repository** (`src/Repositories/QrRepository.php`): Відокремлює логіку запитів до бази даних від коду інтерфейсу.

## 🛠️ Refactoring Techniques (Техніки рефакторингу)
[cite_start]Під час розробки були використані такі техніки[cite: 32, 36]:
* **Extract Method**: Складні блоки коду розбиті на дрібніші приватні методи.
* **Rename Method/Variable**: Назви функцій та змінних змінені на більш описові для покращення читабельності.
* **Self-Encapsulation**: Доступ до даних всередині об'єктів реалізовано через геттери та сеттери.
* **Move Method**: Логіку завантаження медіа було перенесено з контролера у спеціалізований сервіс.
* **Remove Dead Code**: Видалено застарілі коментарі та налагоджувальний код перед фінальним релізом.
